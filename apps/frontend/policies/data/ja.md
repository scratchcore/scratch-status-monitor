# データポリシー

Scratch Status Monitor（以下「本サービス」）は、Scratch サービスの稼働状況を監視・可視化するため、一定の技術データおよび統計情報を収集・保存します。

本ポリシーは、それらのデータの種類、利用目的、保存方針について説明するものです。


## 1. 収集されるデータの種類

本サービスでは、以下のようなデータを収集します。

### 1.1 監視データ

Scratch の公開エンドポイントに対する定期的なリクエスト結果：

- HTTP ステータスコード
- 応答時間（レイテンシ）
- エラー発生有無
- タイムスタンプ

これらはサービスの稼働状況を判断するために使用されます。

### 1.2 システムログ

サービス運用上必要なログ：

- API リクエストログ
- エラーログ
- 実行状況ログ
- Cron 実行履歴

### 1.3 ユーザー関連データ（限定的）

ユーザーがお問い合わせフォームを利用した場合：

- メールアドレス
- メッセージ内容
- 送信日時

## 2. データの利用目的

収集されたデータは、以下の目的にのみ使用されます：

- ステータス判定
- 障害検知および分析
- サービス品質の改善
- セキュリティ対応
- ユーザーサポート

## 3. 保存期間

データは運営上必要な期間のみ保存されます。

目安として：

- 監視データ：最大 12 か月
- システムログ：最大 90 日
- お問い合わせ内容：対応完了後、合理的期間内に削除

保存期間は、運用状況に応じて変更される場合があります。

## 4. データの保存場所

本サービスのデータは主に以下のサービス上に保存されます：

- Supabase（監視データ・問い合わせ情報）
- Cloudflare（アクセスログ・セキュリティログ）

各サービスは、それぞれのセキュリティポリシーに基づいてデータを管理します。

## 5. 個人情報の取り扱い

本サービスは、Scratch アカウント情報や個人識別可能な Scratch データを意図的に収集しません。

監視対象は公開 API および公開エンドポイントに限定されています。

## 6. データの共有

収集されたデータを第三者へ販売・提供することはありません。

ただし、以下の場合を除きます：

- 法令に基づく開示要請があった場合
- サービス運営に必要な範囲で外部基盤へ保存される場合

## 7. データの削除請求

ユーザーは、自身が提供した情報（お問い合わせ内容など）について削除を請求できます。

希望される場合は以下よりご連絡ください：

https://ssm.scra.cc/contact

## 8. プライバシーポリシーとの関係

本ポリシーは[プライバシーポリシー](/policies/privacy)に記載されたプライバシーポリシーの一部を補完するものです。

## 9. ポリシーの変更

本ポリシーは必要に応じて更新される場合があります。  
変更後の内容は本ページにて公開されます。
